Windows EventLog Helpdesk

Version {{VERSION}} · {{RELEASE_DATE}}

L’application charge les canaux actifs de Windows ou des fichiers EVTX/EVT, crée une preuve Markdown complète, lance une analyse locale avec Ollama et permet des questions de suivi basées sur les événements et l’analyse.

Flux de travail

  1. Actualiser l’arbre, sélectionner un journal et le charger.
  2. Choisir une limite; 0 signifie tous les événements.
  3. Utiliser éventuellement un filtre XPath.
  4. Choisir un modèle Ollama et la langue de réponse, puis lancer l’analyse.
  5. Utiliser l’onglet de discussion pour les questions de suivi.

Intégrité

Chaque événement du Markdown complet conserve le résumé, tous les champs XML aplatis et le XML brut original. Le PDF est un résumé dérivé.

Ollama

Le point de terminaison par défaut est http://127.0.0.1:11434. Les requêtes sont exécutées en arrière-plan avec diffusion progressive; l’interface reste utilisable. Le mode automatique segmente les grandes entrées et produit ensuite une synthèse finale.

Confidentialité

Avec le point de terminaison par défaut, les données restent locales. Un point de terminaison distant reçoit le contexte sélectionné.

Contexte système et rapports

L’instantané système est modifiable avant envoi. Les rapports Markdown et PDF automatiques peuvent être désactivés dans les paramètres. Le PDF contient des tableaux et graphiques calculés.
